# Diagnosis
The current dashboard does not track or display the implementation status of user feedback. The feedback items lack a `status` field, and there is no UI component to visualize the status (e.g., pending, in-progress, completed). Additionally, there is no mechanism to automatically update the status when a feedback item is marked as implemented, preventing users from seeing which feedback has been acted upon.

# Proposed change
Modify `/opt/axentx/review-pilot/dashboard.js` to:
1. Add a `status` property to feedback items (with enum values: `pending`, `in-progress`, `completed`, `rejected`).
2. Render a color-coded status badge next to each feedback item.
3. Add a handler to update the status when a feedback item is marked as implemented.
