## 1. Acceptance Criteria
1. **Deterministic Ranking** – Given the same list of feedback items and the same configuration, `rankFeedback` returns an identical ordered array on every execution.  
2. **Configurable Scoring** – Scoring weights for each criterion (e.g., impact, urgency, frequency, severity) are loaded from a JSON configuration file; modifying the file changes the ranking without code changes.  
3. **Priority Labels** – Each item receives a numeric `score` and a derived `priority` label (`Critical`, `High`, `Medium`, `Low`) based on configurable threshold values.  
4. **Filter by Priority** – The exported helper `filterByPriority(list, label)` returns only items whose `priority` matches the supplied label, preserving the original ranking order.  
5. **Graceful Degradation** – Missing or non‑numeric criterion values are treated as `0` (and logged) so the algorithm never crashes and still produces a ranking.  
6. **Performance Bound** – Ranking 10 000 feedback items completes in ≤ 200 ms on a standard production Node 14+ environment (measured with `process.hrtime`).  
7. **Audit Trail** – `rankFeedback` returns an object `{ rankedList, rankingLog }` where `rankingLog` is an array of `{ id, score, breakdown: {criterion: value, …} }` for every processed item.
